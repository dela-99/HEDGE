# Contributing to fintech-secure-app

## Team rules

- All architecture and decision documentation lives in `/docs`.
- Code lives next to its service under `/apps` or in shared packages under `/packages`.
- Never commit secrets, credentials, or environment-specific sensitive values.
- Every big change gets an ADR in `/docs/decision-records`.
- Use branch names in the form `feature/xxx` or `bugfix/xxx`.
- `main` is a protected branch.

## Working agreement

Before starting a significant change, review the relevant files in `/docs/architecture` and update `/docs/onboarding.md` when the developer workflow changes.
